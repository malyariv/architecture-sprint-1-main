# Задание 1. Разделить проект Mesto на несколько микрофронтендов

выполнил только первые 2 пункта

## Проектирование

Для разделения проекта на микрофронтенды был использован Webpack Module Federation, потому что весь код использует React, можно шарить общий код и использовать библиотеки для хранения глобального состояния.

## Структура и описание

Можно выделить три логические части:

- часть, ответственная за авторизацию
- часть, ответственная за работу с контентом пользователя и реакциям
- часть, ответственная за работу с профилем пользователя

Предлагается следующие 4 микрофронтенда:

- **auth** - удаленный модуль аутентификации
- **profile** - удаленный модуль для работы с профилем пользователя
- **ugc** - удаленный модуль для работы с контентом пользователя и реакциям
- **host** - основное приложение, которая загружает модули auth/profile/ugc

# Задание 2. Декомпозиция приложения

## Роли и сервисы

Можно выделить 4 роли:

1. **Продавец | Исполнитель** - размещает товар / услугу на платформе
2. **Покупатель | Заказчик** - заказывает товар / услугу на платформе
3. **Администратор** - проводит валидацию аукциона и заявок на него, получает отчеты
4. **Специалист техподдержки** - рассматривает аппеляции


Можно выделить 9 сервисов с отдельной бизнес-логикой:

1. **Сервис авторизации** - регистрация пользователей, выдача ролей и прав
2. **Сервис оповещений (нотификаций)** - отправка оповещений пользователям. Взаимодействует с другими сервисами через Кафку
3. **Сервис личный кабинет** - операции редактирования товаров и услуг пользователя перед размещением на площадке
4. **Сервис управление заказами и товарами** - размещение товаров / услуг / заказов на платформе. Пользователи в личном кабинете могут написать драфт, потом попробовать разместить на платформе, дальше желательно модерация проверяет, и уже в конце объявление с товаром / услугой / описанием заказа публикуется на платформе.
5. **Сервис аукцион** – операции с аукционом, включая его создание
6. **Сервис поиска** – поиск товаров и услуг на платформе. В дальнейшем можно управлять выдачей, продавая ВАСы
7. **Сервис статистики** – генерация различных отчетов. Можно тоже сделать асинхронным. В любом случае использовать паттерн replica read
8. **Сервис оплаты** – проведение оплаты за товары / услуги
9. **Техподдержка** – обращения в техподдержку
